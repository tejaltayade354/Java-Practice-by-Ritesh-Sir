package collection.list;
import java.util.LinkedList;
public class LinkedList1 {                            //homogeneous data
    public static void main(String[] args ){
        LinkedList ll=new LinkedList();
        ll.add(2000); ll.add(25);ll.add(1000);
        System.out.println(ll);
        ll.add(true);ll.add('x');ll.add(null);          //heterogeneous data, null allowed,

        System.out.println(ll);
    }
}

package collection.list;

import java.util.LinkedList;

public class LinkedList2 {
    public static void main(String[] args){
        LinkedList ll=new LinkedList();
        ll.add(100);ll.add(200);ll.add(300);
        System.out.println(ll);
        //adding 1 linked list into another list  using Constructor based approach
        LinkedList ll2=new LinkedList(ll);
        ll2.add('x'); ll2.add(true);
        System.out.println(ll2);
        //adding using addAll()
        LinkedList ll3=new LinkedList();
        ll3.addAll(ll2);
        ll3.add("ma i"); ll3.add(false);
        System.out.println(ll3);

    }
}

/*
op:
[100, 200, 300]
[100, 200, 300, x, true]
[100, 200, 300, x, true, ma i, false]


 */

- there are 2 constructors supported by Linkedlist class.

1)public java.util.LinkedList()
2)public java.util.LinkedList(java.util.Collection)

-IQ: Linkedlist implements majority of the characteristics of array list, then when should we go for arraylist and when for linked list?
-> whenever there is need to add frequent data to the collection, we will not prefer to go for array list, we will prefer to go for linked list. reason: whenever we try to add any data to the array list , adding 1 data /elent will make the objects perform multiple rightshift operations, due to which it will consume more time to and may degrade the performance. hence in such situations we go for linked list.bxcoz to add 1 object, we will just have to we just have to 1 link between nodes, and it will be less time consuming, hence whenever we have to add multiple data frequently , we prefer to go for linkedlist, rather than array list.

-whenever there are multiple read operations, in such situations, we will prefer to go for arraylist rather than linked list.
-reason is: to extract the data from likedlist, we not only have to extract the node data, but we also have to read the address of the next node, hence not only it consumes more time to read node and address, but bczo of this , it will consume more memory. which reduces the speed of reading or extraction.
- when it comes to array list, we only have to read the obj data. bcoz every obj data is next to each other.hence reading becomes faster.


-Arraylist also implements RandomAccess bcoz of which it is responsible , to get extra power to the array list, to perform extract operation.

public class Vector2 {
