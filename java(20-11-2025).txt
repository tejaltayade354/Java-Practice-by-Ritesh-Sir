package collection.list;

import java.util.ArrayList;

public class ArrayList1 {
    public static void main(String[] args){
        ArrayList al= new ArrayList();
        al.add(100); //int-------> Integer---------> AUTOBOXING
        al.add(200); al.add(300);  //homogenous data is allowed to store in collection
        System.out.println(al);
        al.add('x');al.add(10.2f);            //heterogeneous data is allowed to store in collection
        System.out.println(al);
        al.add(100); al.add(200);                 //duplicate data is also allowed.
        System.out.println(al);
        al.add(null);                             //null is also allowed.
        System.out.println(al);
    }
}

op:
[100, 200, 300]
[100, 200, 300, x, 10.2]
[100, 200, 300, x, 10.2, 100, 200]
[100, 200, 300, x, 10.2, 100, 200, null]

Process finished with exit code 0
 */

package collection.list;
import java.util.ArrayList;
public class ArrayList2 {
public static void main(String[] args){
    ArrayList al=new ArrayList();
    al.add("hp");
    al.add("lenovo");
    al.add("dell");
    for(Object o:al){ ////for loop to extract teh data----> and we write object means we are doing upcasting as string wants to behave as object class

    }


}
}

-example 4:



-as seen in array list 4, the collection can store any type of data( heterogeneous type of data/homogeneous data ) due to which there are 2 major problems encountered.
1) data is not type safe: normal collections cannot store or cannot guaranty only a specific type of data to be stored, which will result in manipulation in the data.
2)because of using heterogenous collections, for every object, we have to check instanceof operation and we have to perform downcasting, due to this , it will increase the length of the code, which will impact on the memory(more consumption) and excetution(slower), hence in real time, we prefer to use collections in a homogeneus way, rather than heterogenous , hence to make the collections homogeneous we use generics.

-generics were introduced in version 1.5V, which made collections more stronger.



package collection.list;

import java.util.ArrayList;

public class ArrayList4 {
    public static void main(String[] args){

        ArrayList al=new ArrayList();
        al.add(new Swiggy(101,"PIZZA"));
        al.add(new RedBus(101,"shampy"));
        al.add(new Emp(101,"Pajji"));
        for(Object o:al){
            if(o instanceof Swiggy){
                Swiggy s=(Swiggy)o;
                System.out.println(s.prodId+" - "+s.prodName );
            }
            else if(o instanceof RedBus){
                RedBus r=(RedBus)o;
                System.out.println(r.ticketId+" - "+r.passengerName );
            }
            else if(o instanceof Emp){
                Emp e=(Emp)o;
                System.out.println(e.empId+" - "+e.empName );
            }
        }

    }
}
/*
op:

101 - PIZZA
101 - shampy
101 - Pajji

*/
