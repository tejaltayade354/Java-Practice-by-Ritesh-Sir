java(08-11-2025)

-whatever members defined in java, which has a name is given a  naming standard in java.

-classes:

- according to java standards , every class should start with a capital letter.
- if a class contains , 2 or more words then each word within the class, the starting letter should be capital.
- examples:
1) Demo/Sample- user defined classes 
2) String/StrinBuffer - predefined classes
3)FileNotFoundException

-constructors:
- constructor name should be same as class name.
-this convention followed by java comes under pascal convention.

-interfaces:
-it also follows same convention as classes.
-Examples: 1) Clonable
           2) AttributedCharacterIterator

- methods;
-If a mmethod contains single word, then the starting letter should be in lowercase.
-If a mmethod contains  more than one word, then the starting letter of the 2nd word onwards should be in uppercase.
-examples:
1)main, trim, concat i lower case
2)toString();
3)toUpperCase()


- this approach or convention is from camel Case convention.

-variables:
- follows same convention as methods.

-packages:
-every package whatever we create should be in lower case.
-examples:
1) java.lang
2) java.io
3) java.util

-constants:

- every letter within a constant is represented  in a upper case.
- if a constant contains 2 or more words , this words are connected with a underscore character("_").
-examples:
1)BYTES
2)MAX_VALUE
