POLYMORPHISM:
- POLY-many, multiple
- MORPHISMS- forms, behavior.


-an object having more than 1 behavior , this process comes under polymorphism.


POLYMORPHISM can be categorized into 2 types-
1) compile time/static binding/early binding polymorphism
2) runtime/dynamic /late binding polymorphism.

1) compile time/static binding/early binding polymorphism:
- compile time polymorphism can be achieved with the help of overloading.
- overloading can be categorized into 3 types:  1)method overloading          2)constructor overloading    3)operator overloading.

1)method overloading:


- RULE1:
- to achieve method overloading, method name should remain the same, but should vary in NO OF ARGUMENTS.

-example1: rule 1 satiisfy

 class demo{
    void m1()
    {  
          System.out.println("inside m1  method 0 arg ");
}
    void m1(int a)
    {    
        System.out.println("inside m1 method  1 arg");
}
 }
 class example1 {
    public static void main(String[] args){
    demo d=new demo();
    d.m1(10);
    }
}

3) OPERATOR OVERLOADING:
- Java does not support operator overloading except the '+' operator. The plus operator either perform addition or concatenation.
- the plus operator will perform addition, if both the inputes to the operator is numeric.
- the plus operator will behave as concatenation , if both or single input contains a string argument.

example12:


class example12 {
    public static void main(String[] args){

    
    System.out.println("OPERATOR Overloading CONCEPT");
    System.out.println(70 +70);         // addition
    System.out.println(70+ "start");           // NUM+STRING->CONCATINATION
    System.out.println("mysuccess"+ "start");           // string+STRING->CONCATINATION


}}
