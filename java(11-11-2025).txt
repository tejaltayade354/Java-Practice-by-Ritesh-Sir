-Wrapper classes:
-Wrapper classes are used to convert primitive data in the form of objects.

q1)What is the need for wrapper classes?

->To increase a security level of each data member, we go for wrapper classes.

imp note:
-based on total no of primitive data types that many no of wrapper classes are supported by java.

-primitive data type- wrapper class

-all wrapper classes come under java.lang package.

-How do we convert primitive data in the form of object?
-> We can convert using constructor based approach- there also 2 constructor approaches one for int AND for string

public java.lang.Integer(int);

IQ: how do we convert a wrapper object  to primitive data type?
->  using xxxxxValue() method: xxxxx represents int(xxx), float, double short etc


class Example7{
    public static void main(String[] args){
        Integer i1=new Integer(100);
        Integer i2= i1.intValue();     //xxxxValue()-> to convert wrapper obj to primitive
        System.out.println(i2);      
        System.out.println(i2.toString());     //invalid, but why giving output.                    
     }}

     op;

     100
     100

- Autoboxing converts i2 into an Integer.
- Integer.toString() returns "100".
