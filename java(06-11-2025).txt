-checked exception can be delegated or can be handled, unchecked exceptions can only be handled, cannot be delegated.


-exceptions can be generated during the compile time or runtime, but they can only be handled during runtime.

- what is the difference between , try-catch block and throws keyword?
-> the try-catch block:
1) used to handle the exception.
2) try catch is a block.
3) try-catch can be used at ana implementation level.
4) it is used within method, constructor and blocks.
5) it can generate difft types of exception, but at any point of time , it can generate only 1 exception.

the throws keyword:
1) it delegates the exception.
2) throws is a keyword.
3) but throws keyword can be used at declaration level.
4) throws keyword can be used at method and constructor level .
5) throws  keyword can throw multiple exceptions at a time.

- the throw keyword:


-the throw keyword,  is responsible to customize predefined exception info.
