example 10
-constructors are never stored in a heap area. They are only executed during the object creation ,within the stack area.

class Employe{
    int empid;
    String empname;
    Employe(int empid,String empname)   //String always written with capital S
    {
        this.empid=empid;
        this.empname=empname;
    }
    void displayinfo()
    {
        System.out.println("The Employe id is " +empid+ " and name is " +empname  );
    }
    
}
class example10 {
    public static void main(String[] args){
        Employe e1=new Employe(101, "JAGGU JASOOS");  //constructors are always executed during object creation.
        e1.displayinfo();
        Employe e2=new Employe(102, "KALIYA BOSS");
        e2.displayinfo();
    }
}

O/P

The Employe id is 101 and name is JAGGU JASOOS
The Employe id is 102 and name is KALIYA BOSS

-hence by creating one object, it is possible to execute multiple  constructor within the same class using this keyword.
-also it holds the address of the current object .

     example12(){
        this(24);

}
  example12(int a){
    System.out.println("inside 1 argument constructor");
}

example12 e1=new example12(); 
}
}

//this()--->statement/method is used to call another constructor within the same class.
