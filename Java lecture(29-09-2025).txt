IQ: both constructors and non static blocks are executed during object creation, BUT out  of both which one will get executed first?
--> First nonstatic blocks will get executed first and only then constructors get executed.


    
During object creation, whenever the control is given to the constructor, the constructor first checks whether any instance blocks are present within a respective class, if they are present then it will execute the instance block first. And only after that, it will then execute the constructors.

EXAMPLE8:

-instance blocks are always executed in the stack memory.
-instance blocks are never saved in heap memory, they are only called during object creation.

IQ: When instance block and constructors perform the same operation then what's the difference?
->  The instance block logics are applicable to all objects and constructors, but constructor logics are applicable only to the respective object as well as to respective constructor.


 class example8 {
    int a;

    {
        System.out.println("Inside nonstatic bloack-a :" +a);
        a=10;
    }
    example8(){
        System.out.println("before Inside 0 arg constructor a: " +a);
    
        a=100;

        System.out.println("afterr Inside 0 arg constructor a: " +a);
    }

       example8(int x){                                                      // constructor logics are only applicable to respective constructor, and object.
        System.out.println("before Inside 1 arg constructor a: " +a);
    
        a=x;

        System.out.println("afterr Inside 1 arg constructor a: " +a);
    }
    public static void main(String[] args){
        example8 e1=new example8();
        example8 e2=new example8(1000);
    }
}
