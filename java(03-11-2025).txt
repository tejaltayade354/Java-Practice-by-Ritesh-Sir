IQ: what do u mean by fully checked exception  and partially checked exception?
--->

1) FULLY CHECKED EXCEPTION.
- If the root exception is checked, and all its child exception is also checked , there is such n exception called as fully checked exception.
-example: IO exception
- below that we have partially checked exception

2) partially checked exception.
- if the root class or an exception class contains some child class checked and some unchecked , such exceptions are called as partially checked exception.

-example: throwable(C), exception

EXCEPTION HANDLING:
-Exception handling involves 1) the program should have normal termination.
                             2) whenever a exception is handled, all statements should be executed.

- How many ways are there to handle the exception?
-> there are 2 ways to handle the exception:
 1) Using the try-catch block
 2) Using throws keyword


Try-catch block:

-imp note:

-Whichever statements throw or raise an exception, those statements should be part of TRY block.
-the alternative solutions should be part of CATCH block.


-syntax:

 try
  {
    statements->generate/raise an exception.   
}
catch(type of exception generated)
{
alternative solution statements
}

-example4:
-using the try catch block, we handle the exception.
-inside try block- whenever exception occurs the control will go to the catch block.
- And in catch block , alternative solution is to be provided.



 class example4 {
    public static void main(String[] args){
        System.out.println("starttttttttttttttt");
        try{
       System.out.println("inside try block");
       System.out.println(10/0);          ///exception occur-------> object created (rhs of obj creation)
        }

  System.out.println("inside catch block");
       System.out.println(10/5);

}

-Whenever we say, an exception is generated, internally it means an object gets created and that object address will be stored in the ref variable of the catch block.

-if an exception is not generated within try block, then the control will never go from try to catch block.
