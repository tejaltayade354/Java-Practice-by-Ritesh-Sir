3)Anonymous classes:
-They are also called as nameless classes.
-Anonymous class helps to partially reduce the code length.

-example 18:
-as seen in example 18, it doesn't matter in which class u are implementing the methods, hence the implementation can occur in any of the classes

abstract class Demo{
    abstract void m1();
    abstract void m2();
}
class Sample extends Demo{
     void m1(){
        System.out.println("inside m1");     // consumes tmore lines, hence we use anonymous class, bcoz user only wants implementation of methods , class name is not imp.
     }
     void m2(){
        System.out.println("inside m2");
     }
}
class Example18 {
    public static void main(String[] args) {
       Sample s= new Sample();
       s.m1();
       s.m2();
    }
}

-example 19:

-unknown classes or anonymous classes means, we as a developer or automation tester may not give the name but java compiler internally may give the name. Such classes are technically are called as Anonymous classes.

- ex 19, line no 7:static Sample s=new Sample()
this line performs multiple tasks:
1) it will represent to connect the anonymous class implementation with the parent class  sample.
2) it is also responsible to load the anonymous class object creation in a memory and store the reference in 's'.
3) the respective line do not represent the object creation of abstract class.

abstract class Sample{
    abstract void m1();
    abstract void m2();
}
class Example19 {
     
    static Sample s=new Sample() {        //anonymous class object
     void m1(){
        System.out.println("inside m1");     // consumes tmore lines, hence we use anonymous class, bcoz user only wants implementation of methods , class name is not imp.
     }
     void m2(){
        System.out.println("inside m2");
     }
    };
    public static void main(String[] args) {
    s.m1();
    s.m2();
       
    }
}
