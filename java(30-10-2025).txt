-IQ: What is the difference betn string buffer and stringbuilder?
---> the string buffer and string builder class are almost same, but the difference is the string buffer methods are synchronized, and string builder methods are not synchronized.

-if a method is synchronized, one thread is able to access one method at any point of time, transactions are safe, but performance is a little lower.
- if the methods arenot synchronized, multiple threads are able to access the methods, transactions are not safe, but performance is better.(threads are customer, methods are cashier) 






STRING TOKENIZER CLASs:

-It is part of java.util package.
-String Tokenizer is primarily used to split a string in the form of tokens.

-A TOKEN can be defined as smallest element within the string is called as Token.


- the string tokenizer objects are created using new keyword.

-example27:
- whenever we using string tokenizer, we have to inform compiler java,  it  is present in java.util. AND  to to do that we have to use a statement import.

- the JVM during the execution, will import the respective class, within the memory and use it for spitting the string.
- the string tokenizer is responsible to break the string in the form of tokens. and to access this tokens we need to perform 2 steps, 1) we check token element is present or not, If it is present, we will extract the tokens. this can be achieved by 2 methods- hasMoreElments(String);   and nextElement();


- the default delimeter, to split the string is by default is space.


import java.util.StringTokenizer;

class ex27 {
    public static void main(String[] args) {
        String s1="this is string tokenizer example";
        StringTokenizer st=new StringTokenizer(s1);
        while(st.hasMoreElements()){          //checks tokens present or not.----> //true
            System.out.println(st.nextElement());          //extract the tokens from string
        }
    }
}
