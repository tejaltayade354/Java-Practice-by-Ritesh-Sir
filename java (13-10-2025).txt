-example5:

-an abstract class can contain both complete as well as incomplete methods, hence it is said that abstract class are not 100% abstract.

-The abstract class is not allowed to create an object, but it(abstract reference) can hold the address of the child object.

abstract class sample{
   abstract void  m1();
   void m2(){
    System.out.println("inside m2 method");
   }
} 
class demo extends sample
{       void m1(){
    System.out.println("inside m1 method");
}

}
class example5 {
    public static void main(String[] args){
        sample d=new demo();
        d.m1();
        d.m2();
        
    }
}
/* 
op:

inside m1 method                                                                                                
inside m2 method

-Example 8:

IQ:
-we can create variables within a abstract class.(leave inheritance and polymorphism for this example.)

-variables are not part of abstraction.

class sample
{
    int a=45;      //variablles are also declaredd in abstract class, but they are not part of abstraction.
}
class demo extends sample
{
    int b=5;
}
class example8 {
    public static void main(String[] args) {
        //upcasting
        sample s=new demo();
          System.out.println(s.a);      //45
          //downcasting
          demo d=(demo)s;
          System.out.println(d.b);           //5
        System.out.println(d.a+d.b);               //print 50
    }
}

45
5
50


-example 9:
-constructors are allowed within abstract class.


abstract class sample{
    sample(){
        System.out.println("inside 0 argument sample constructor");
    }
}
class demo extends sample{
    demo(){
        super();            // to call parent class constructor
        System.out.println("inside 0 argument demo constructor");
    }
}
class example9 {
    public static void main (String[] args){
 demo d=new demo();
}
}

inside 0 argument sample constructor
inside 0 argument demo constructor

IQ: difference between normal & abstract classs.

NORMAL CLASS-
1) we can create an object.
2) In a normal class, we can only create complete methods .

ABSTRACT CLASS-
1)In abstract class, we CANNOT create an object.
2) In abstract class, we can create an complete and incomplete method too.
